Examen Abril 2026

Proyectos y carpetas presentes en el repositorio del examen:

- data: contiene un fichero CSV con datos de clientes, sus pedidos y los productos de los pedidos de una tienda de informática
- android: cliente fork del almacen para acceder a los datos del backend "tienda"
- importDataClient: proyecto Java-Gradle para intellij que se encargará de leer los datos del CSV y "enviarlos" al servidor utilizando peticiones GET con Okhttp y la API definida en los servlets del backend
- servlets: proyecto Java Maven J2EE con servlets para servidor Tomcat 9 con la estructura para realizar el backend de la tienda
- infraestructura: script docker-compose y de BBDD mysql para crear las máquinas virtuales necesarias para la implementación del backend

Objetivos:

Ejercicio 1. CSV to BBDD
Realizar el cliente de importación de datos en Java. Completar el proyecto de la carpeta importDataClient, de manera que este lea el fichero CSV, lo convierta a datos del modelo de la tienda (Clientes, Pedidos, LineaPedido, Productos) y los envíe a la BBDD asociada al backend utilizando el servlet importData. Podéis estructurar este proyecto como queráis, creando una vista "mínima", pero es obligatorio usar OKHttp como librería de trabajo para enviar los datos leídos del fichero, y se aconseja mandar los datos en JSON al servidor para que el servlet lo procese. Para ello, tenéis que utilizar el siguiente código en el cliente:

OkHttpClient client = new OkHttpClient();

String json = "...tu json...";

RequestBody body = RequestBody.create(
    json,
    MediaType.parse("application/json")
);

Request request = new Request.Builder()
    .url("http://midominio.com/importar")
    .post(body)
    .build();

Response response = client.newCall(request).execute();


En este caso, como es un POST, y no un GET, no hay que "esperar" nada. Por lo que no hay que "encolar" ni crear una Callback. Para ver que funciona, nos vamos a la BBDD y vemos si están los datos.

EVALUACIÓN: 
- Lectura del fichero: 2p
- Transformación (parser) del CSV a modelo: 2p
- Envío de datos al servidor: 2p

Ejercicio 2. Server
Completa el proyecto de servlets para construir los puntos de acceso (tantos servlets como puntos de acceso) descritos en el interface DataAccess del package controller.

 - public String listAllProductos(); -> Accede a la BBDD y obtiene todos los productos
 - public String findProductoXCodigo(String code); -> Accede a la BBDD y busca el producto que coindice con codigo=code
 - public String findClienteXDNI(String dni); -> Accede a la BBDD y busca el cliente con el DNI=dni
 - public String listProductosXPedido(String dni, String pedido); -> Accede a la BBDD y busca todos los productos existentes en el id_pedido=pedido y para el cliente DNI=dni
 - public String importData(String jsonDataFromCSV); -> parsea el json con los datos del CSV leído en el cliente y realiza los INSERT adecuados para actualizar la BBDD con la info almacenada en ese JSON

IMPORTANTE: Este último método tiene que llamarlo un servlet que implemente el método doPost, si no no funcionará. Para "recoger" los datos del json, al ser un parámetro POST, en el servlet, tenéis que usar el siguiente código:

BufferedReader br = request.getReader();

String linea;
StringBuilder sb = new StringBuilder();

while((linea = br.readLine()) != null){
    sb.append(linea);
}

String json = sb.toString();

EVALUACIÓN:
- Cada servlet con su método del controlador y con su acceso a BBDD: 2p
- El servlet de importación: 6p

Ejercicio 3. Cliente Android
Usando de base el proyecto de la app del almacen, cambia la app para que muestre una lista en la MainActivity con los clientes existentes en la BBDD, o con los productos existentes en la BBDD. Para ello, quita el "formulario" de introducción de nuevos productos del almacen y deja dos botones, uno llamado "Clientes" y otro "Productos", de manera que según se pulse uno u otro, se muestren uno u otro listado.
